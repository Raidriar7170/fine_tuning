本阶段改动

  • 新增 assistant target span 解析：目标 JSON 必须在 rendered SFT text 中唯一出现；重复或缺失时 fail closed。
  • 新增 assistant-only label 构造：system/user/role 前缀和其他非目标 token 标为 -100，只有 assistant Browser Task Contract JSON token 保留 loss。
  • 默认 runtime objective inspector 不再使用 full-text labels；它使用同一套 assistant-only label path，并保留 real-training provenance gate。
  • 真实 SFT heavy path 保持 TRL SFTTrainer，但改为 pretokenized input_ids/attention_mask/labels，避免 SFTTrainer(dataset_text_field="text") 静默训练 prompt tokens。
  • README 和 SFT metadata 增加 loss-mask policy，说明 SFT 仍使用 TRL/PEFT 路径，但改为 pretokenized assistant-only labels；DPO 仍使用 TRL preference path。
