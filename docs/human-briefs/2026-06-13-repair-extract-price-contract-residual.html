extract-price 残差修复阶段简报

一句话结论：7B A100 训练让 extract-price 不再走 search，也不再用 query/page_url slot，但 strict exact 仍是 0/3，主要卡在 canonical 文案和 target 词面。

当前状态

阶段状态：archived；证据类型：public-sample train-split diagnostic；不是 dev/test 泛化、生产可用、checkpoint/adapter release 或 live-browser benchmark 结论。

本阶段改变

  • 新增 extract-price prompt policy 和两类 DPO hard negative：search fallback、query/page_url slot drift。
  • public DPO 从 46 对更新到 48 对。
  • 在 A100 GPU 3 上跑 7B SFT train-split rerun，并只回传 sanitized evidence。

剩余风险

该结果仍是 train-split public sample 证据。它支持“任务/路由形状学会了”，不支持“strict extract canonical target 已学会”。

推荐下一步

开一个更小阶段专门修 canonical extract target wording：让“多少钱/标价/页面上的商品价格”统一落到 slots.target="商品价格" 与 normalized_command="提取页面商品价格"，先不要扩大到私有全量。
