  • 从已提交的 candidate-design JSON 读取 3 个 clarify boundary themes。
  • 每个 theme 生成 1 条 train、1 条 dev、1 条 test seed，共 9 条 standalone candidate seeds。
  • 每条 seed 派生 original + 2 augmentations，共 27 条 SFT candidate rows。
  • 写出 JSON evidence、Markdown evidence、manifest、SFT sidecar 和 leak-scan result。
  • 更新 CONTEXT.md 与 reports/final_status.md，避免继续说“还没 materialize”。

  • 没有训练、没有 prediction，所以不能宣称 strict slot F1 或 exact match 变好。
  • 后续如果合入 formal public sample，会形成新的 manifest boundary，旧 metrics 不能直接比较。
